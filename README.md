# How to Use:
This repository uses feature branches for each member. Do not commit directly to the main branch. (Correct me if I'm wrong/if you have a better way of handling this TwT not an expert at using this)

- Do not commit directly to the main branch.
- Each member has their own assigned branch. You can choose to edit other files if necessary (just indicate sa changelog)

1. Open the [repository](https://github.com/jpfacto/valorantFinalProject).
2. Switch to your assigned branch.
3. Click Add file and then Upload files.
4. Upload the files assigned to you.
5. Enter a descriptive commit message. (ex: Added player.cpp and player.h)
6. Select Commit directly to the current branch.
7. Click Commit changes.
8. Create a Pull Request by going [here](https://github.com/jpfacto/valorantFinalProject/pulls)
9. Click New Pull Request
10. Check that the base is main and that the compare is your branch (ex: for Janelle, base: main; compare: player)
11. Click Create Pull Request
12. Add the following inputs: (Example)
Title: Add Player Module
Description:
- player.cpp
- player.h
- main.cpp
13. Click Create Pull Request
14. Janelle will see it then approve the merge (it would be merged into the main branch (main version of the code))
15. Go to the main branch
16. Update CHANGELOG.md by adding a new entry describing your changes. (See Janelle and John's examples)
17. Update in the Messenger groupchat tooo
